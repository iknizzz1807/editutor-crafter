## Core Philosophy: The Socratic Journey
1. **No Jargon Left Behind**: Never use a technical term (TCP, Heap, Stack, Pointer, Port, etc.) without a "Quick Breakdown" box or an intuitive analogy.
2. **Mental Models First**: Every section must start with a powerful analogy before showing a single line of technical specification.
3. **The Microscope Effect**: Zoom in! If you mention an object in memory, describe its bits and bytes. If you mention a network packet, describe its headers.
4. **Pedagogical Anchors**: Every theoretical explanation must immediately lead to a visual diagram marker `{{DIAGRAM:id}}` and then to a practical code scaffold.

## Writing Rules
- Use a professional yet encouraging tone.
- Target high information density: no fluff, only deep technical truth explained simply.
